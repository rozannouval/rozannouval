## Engineering Principles

> I approach every project with the same discipline — understand the problem deeply before writing a single line of code.

| Principle | Practice |
|---|---|
| **Clarity over cleverness** | Code should communicate intent, not demonstrate complexity |
| **Design before implementation** | Architecture decisions made upfront save weeks of refactoring |
| **Own the full lifecycle** | From API contracts to UI states — the full picture matters |
| **Iterate with intent** | Ship early, gather signal, improve continuously |
